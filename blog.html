[1] Getting your first $250 dollar bounty: Reflected XSS

A throw back to my first ever bug caught out of blue while checking out the current status of an item I bought which was sent via a well established Logistics company.

Vulnerability Overview:

The identified vulnerability is a reflected Cross-Site Scripting (XSS) vulnerability. It allows an attacker to inject malicious code into the website's HTML response, which is then executed by the victim's browser. In this case, an attacker could execute arbitrary JavaScript code by manipulating the "trackNo" parameter.

Vulnerability Impact:

Exploitation of this vulnerability can have several negative consequences, including: Execution of arbitrary JavaScript code within the context of the victim's browser, leading to potential theft of sensitive information, session hijacking, or further attacks. Compromised user experience and trust due to unexpected or malicious behavior on the website. Damage to the reputation and credibility of Logistics site.

Steps to Reproduce:

To reproduce this vulnerability, follow these steps: Access the URL: https://www.logisticsamplesite.com/path/to/xss?param1=0&&trackNo=%3Cscript%3Ealert(1)%3C/script%3E. Observe that the injected script is executed, resulting in the display of an alert dialog with the message "1".


Recommendation:

To mitigate this vulnerability, we recommend the following actions: Implement proper input validation and sanitization techniques to ensure that user-supplied data is properly encoded or filtered to prevent the injection of malicious code. Implement Content Security Policy (CSP) headers to restrict the execution of untrusted JavaScript code. Conduct a thorough security review and penetration testing of the entire website to identify and address any additional vulnerabilities.

[2] Getting your $180 dollar bounty: Stored Cross-Site Scripting (XSS) via Login Page Notice Banner

A throw back to one of my first ever bug caught out of blue while checking authenticated users ability to modify contents reflected in the login page.

Vulnerability Overview:

A Stored XSS vulnerability was discovered in the application where a user with privileges to edit the Login Page Notice (displayed on the login page) injected malicious JavaScript code using the following payload: 

<a href="javascript:alert(String.fromCharCode(88,83,83))">Click Me!</a>

When other users visit the login page and click on the malicious link, arbitrary JavaScript executes in their browser in the context of the application, leading to XSS exploitation.

Vulnerability Impact:

• User session hijacking: Attackers can steal session cookies or tokens.

• Phishing and social engineering: Malicious scripts can redirect users or display fake forms.

• Credential theft: Scripts can intercept usernames/passwords on login pages.

• Browser exploitation: Attackers can fingerprint the user or exploit browser vulnerabilities.

• Brand and legal damage: Users affected on a public-facing login screen erodes trust and may have regulatory implications (e.g., GDPR).

• The fact that the payload persists and is visible on a public login page magnifies the risk, especially if users are not authenticated yet and may not be expecting dynamic or interactive content.

Steps to Reproduce:

• Login as a user with permission to edit the Login Page Notice.

• Navigate to the configuration menu which has the feature to edit the Login Page Notice.

• Insert the following payload: <a href="javascript:alert(String.fromCharCode(88,83,83))">Click Me!</a>

• Click on submit to save the changes.

• Now, access the login page.

• You will see the "Click Me!" text displayed.

• Clicking on it will trigger a JavaScript alert popup.



Recommendation:

To mitigate this vulnerability, we recommend the following actions:

• Implement proper input validation and sanitization techniques to ensure that user-supplied data is properly encoded or filtered to prevent the injection of malicious code.

• Implement Content Security Policy (CSP) headers to restrict the execution of untrusted JavaScript code.

• Conduct a thorough security review and penetration testing of the entire website to identify and address any additional vulnerabilities.
